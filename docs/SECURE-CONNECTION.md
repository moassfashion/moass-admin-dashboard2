# অ্যাডমিন ড্যাশবোর্ড ও স্টোরফ্রন্ট সুরক্ষিত সংযোগ

অ্যাডমিন ড্যাশবোর্ড (এই প্রজেক্ট) এবং আপনার স্টোরফ্রন্ট প্রজেক্ট সুরক্ষিতভাবে কিভাবে কানেক্ট করবেন তার চেকলিস্ট ও স্টেপ।

---

## ১. সংযোগ মডেল (কি কি কে কল করে)

```
[স্টোরফ্রন্ট সাইট]  ----শুধু পাবলিক API---->  [অ্যাডমিন অ্যাপের /api/ecommerce/*]
       (shop.example.com)                              (admin.example.com)

[অ্যাডমিন ড্যাশবোর্ড]  ----লগইন করে প্রটেক্টেড API---->  [অ্যাডমিন অ্যাপের /api/admin/*]
       (admin.example.com, same origin)                   (সেশন কুকি দিয়ে)
```

- **স্টোরফ্রন্ট** শুধু **`/api/ecommerce/*`** কল করবে – প্রোডাক্ট, অর্ডার, ক্যাটাগরি ইত্যাদি। এগুলো পাবলিক; লগইন লাগে না।
- **অ্যাডমিন ড্যাশবোর্ড** শুধু অ্যাডমিনরা ব্যবহার করে; লগইন করলে সেশন কুকি দিয়ে **`/api/admin/*`** কল হয়। স্টোরফ্রন্ট থেকে অ্যাডমিন API কল করা উচিত না (এবং ব্রাউজার অন্য সাইটের কুকি পাঠায় না বলেই অ্যাডমিন API স্টোরফ্রন্ট থেকে কাজও করবে না)।

সুরক্ষিত সংযোগ মানে: **স্টোরফ্রন্ট শুধু ইকমার্স API ব্যবহার করবে, অ্যাডমিন ক্রেডেনশিয়াল কখনো স্টোরফ্রন্টে না যাওয়া, এবং প্রোডে CORS + HTTPS সঠিক সেটআপ।**

---

## ২. সুরক্ষিত সংযোগ চেকলিস্ট

### অ্যাডমিন ড্যাশবোর্ড প্রজেক্টে (এই রিপো)

| নং | কাজ | কিভাবে |
|----|-----|--------|
| ১ | প্রোডাকশনে HTTPS চালু রাখুন | হোস্টিং/Vercel/যেখানে ডিপ্লয় করছেন সেখানে SSL চালু থাকলেই হবে। |
| ২ | CORS শুধু স্টোরফ্রন্ট ডোমেইনে দিন | প্রোডে `STOREFRONT_ORIGIN` env ভেরিয়েবল সেট করুন (নিচে স্টেপ ৩)। `*` না দিলে অন্য সাইট থেকে ব্রাউজার API কল করতে পারবে না। |
| ৩ | `AUTH_SECRET` সেট করুন | প্রোডে লগইন সেশন সুরক্ষিত রাখতে বাধ্যতামূলক। |
| ৪ | অ্যাডমিন লগইন URL গোপন রাখুন | অ্যাডমিন ড্যাশবোর্ডের URL পাবলিকলি শেয়ার না করলেই ভালো (যেমন `admin.yourstore.com`)। |

### স্টোরফ্রন্ট প্রজেক্টে

| নং | কাজ | কিভাবে |
|----|-----|--------|
| ১ | API বেস URL এনভায়রনমেন্ট থেকে নিন | কখনো অ্যাডমিন URL হার্ডকড করবেন না। `NEXT_PUBLIC_API_BASE_URL` বা সমতুল্য env ব্যবহার করুন। |
| ২ | শুধু ইকমার্স API কল করুন | শুধু `/api/ecommerce/*` – অ্যাডমিন API (`/api/admin/*`) বা লগইন এন্ডপয়েন্ট স্টোরফ্রন্ট ফ্রন্টএন্ড থেকে কল করবেন না। |
| ৩ | অ্যাডমিন পাসওয়ার্ড/সেশন স্টোরফ্রন্টে রাখবেন না | স্টোরফ্রন্ট কোড বা env-এ অ্যাডমিন লগইন ক্রেডেনশিয়াল দেবেন না। |
| ৪ | প্রোডে HTTPS ব্যবহার করুন | স্টোরফ্রন্ট সাইটও HTTPS এ হোস্ট করুন। |

## ৩. অ্যাডমিন প্রজেক্টে CORS সেট (প্রোডে শুধু স্টোরফ্রন্ট ডোমেইন)

এখন অ্যাডমিন অ্যাপে ইকমার্স API-র জন্য CORS হেডার আছে। **প্রোডাকশনে** শুধু আপনার স্টোরফ্রন্ট ডোমেইন থেকে কল যাতে আসে সেজন্য:

### স্টেপ ১: এনভায়রনমেন্ট ভেরিয়েবল

অ্যাডমিন ড্যাশবোর্ডের প্রোড env (Vercel/Hostinger/যেখানে ডিপ্লয়):

```env
# আপনার স্টোরফ্রন্ট সাইটের URL (প্রোটোকল সহ, শেষে / ছাড়া)
STOREFRONT_ORIGIN=https://shop.yourstore.com
```

লোকাল ডেভেলপমেন্টের জন্য খালি রাখলেই হবে – তখন CORS `*` থাকবে (সব অরিজিন থেকে কল যাবে)।

### স্টেপ ২: next.config এ ভেরিয়েবল ব্যবহার

`next.config.ts` এ ইতিমধ্যে CORS আছে। প্রোডে শুধু স্টোর ডোমেইন দিতে env ব্যবহার করুন:

```ts
// next.config.ts
import type { NextConfig } from "next";

const storefrontOrigin = process.env.STOREFRONT_ORIGIN || "*";

const nextConfig: NextConfig = {
  async headers() {
    return [
      {
        source: "/api/ecommerce/:path*",
        headers: [
          { key: "Access-Control-Allow-Origin", value: storefrontOrigin },
          { key: "Access-Control-Allow-Methods", value: "GET, POST, OPTIONS" },
          { key: "Access-Control-Allow-Headers", value: "Content-Type" },
        ],
      },
    ];
  },
};

export default nextConfig;
```

- লোকালে `STOREFRONT_ORIGIN` না থাকলে `*` → যেকোনো অরিজিন থেকে ইকমার্স API কল করা যাবে।
- প্রোডে `STOREFRONT_ORIGIN=https://shop.yourstore.com` দিলে শুধু সেই ডোমেইন থেকে ব্রাউজার API কল করতে দেবে।

---

## ৪. স্টোরফ্রন্টে API বেস URL (সুরক্ষিত ভাবে)

স্টোরফ্রন্ট প্রজেক্টে:

```env
# .env.local (Next.js) বা .env
# লোকাল: অ্যাডমিন অ্যাপ যেখানে চালাচ্ছেন (যেমন http://localhost:3000)
NEXT_PUBLIC_API_BASE_URL=http://localhost:3000

# প্রোড: অ্যাডমিন ড্যাশবোর্ডের পাবলিক URL (HTTPS)
# NEXT_PUBLIC_API_BASE_URL=https://admin.yourstore.com
```

কোডে শুধু এই ভেরিয়েবল ব্যবহার করুন; অ্যাডমিন URL কোডে লিখে রাখবেন না।

---

## ৫. কি কি কল করবেন / করবেন না

### স্টোরফ্রন্ট থেকে শুধু এগুলো

- `GET /api/ecommerce/products`
- `GET /api/ecommerce/products/[id]`
- `GET /api/ecommerce/categories`
- `GET /api/ecommerce/banners`
- `GET /api/ecommerce/shipping`
- `POST /api/ecommerce/coupons/validate`
- `POST /api/ecommerce/orders`
- `GET /api/ecommerce/settings`

### স্টোরফ্রন্ট থেকে করবেন না

- `/api/admin/*` – অ্যাডমিন API (স্টোরফ্রন্ট ইউজারদের অ্যাক্সেস দেবেন না, এবং ব্রাউজার অন্য ডোমেইনের অ্যাডমিন কুকি পাঠায় না)।
- স্টোরফ্রন্ট কোড বা env-এ অ্যাডমিন লগইন ইমেইল/পাসওয়ার্ড বা সেশন রাখা।

---

## ৬. একই ডোমেইনে থাকলে

যদি অ্যাডমিন ও স্টোর একই ডোমেইনে থাকে (যেমন `yourstore.com/admin` ও `yourstore.com/store`), তাহলে স্টোরফ্রন্ট রিলেটিভ পাথ ব্যবহার করবে: `/api/ecommerce/products`। একই অরিজিন বলে CORS ইস্যু হয় না; তবু প্রোডে পুরো সাইট HTTPS এ রাখুন।

## ৭. সংক্ষিপ্ত সারাংশ

| জিনিস | করণীয় |
|--------|--------|
| অ্যাডমিন প্রটেকশন | ইতিমধ্যে আছে – `/api/admin/*` সেশন দিয়ে প্রটেক্টেড। |
| স্টোরফ্রন্ট API | শুধু `/api/ecommerce/*` ব্যবহার করুন; ক্রেডেনশিয়াল স্টোরফ্রন্টে নেই। |
| প্রোড CORS | অ্যাডমিন প্রজেক্টে `STOREFRONT_ORIGIN` সেট করে শুধু স্টোর ডোমেইন অলাউ করুন। |
| HTTPS | অ্যাডমিন ও স্টোর দুটো প্রোডে HTTPS এ চালান। |
| URL ও সিক্রেট | API বেস URL ও অ্যাডমিন URL env থেকে নিন; অ্যাডমিন পাসওয়ার্ড/সেশন স্টোরফ্রন্টে রাখবেন না। |

এই নিয়ম মেনে সংযোগ করলে অ্যাডমিন ড্যাশবোর্ড ও স্টোরফ্রন্ট সুরক্ষিতভাবে কানেক্ট থাকবে।
